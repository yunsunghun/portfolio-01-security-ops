# 윤리적 모의 침해(Red Team) — Lab 전용 기록 가이드

## 1. 반드시 지킬 것 (법·윤리)

- **허가된 대상만**: 본인 소유 장비, 본인 클라우드 샌드박스, **명시적 허가서가 있는** 범위, CTF/교육 과제 환경.
- **허가 없는** 회사·개인·공공 시스템에 대한 스캔·침투 시도는 **불법**이 될 수 있으며, 이 저장소는 그러한 행위를 **지원하지 않습니다**.
- 포트폴리오에는 **공격 재현 절차**(익스플로잇 단계·페이로드)보다는 **범위·방법론·결과 요약·재발 방지**를 올립니다.

## 2. Lab 범위서(ROE) 템플릿 — 복사 후 사용

| 항목 | 내용 |
|------|------|
| 일자 | YYYY-MM-DD |
| 책임자(본인) | |
| 대상 자산 | 예: `lab-01` VM, `10.60.0.0/24` (가상 예시) |
| 금지 행위 | 스코프 외 IP, 데이터 유출, 서비스 거부 공격 등 |
| 중단 조건 | Lab 외부 영향 징후 발견 시 즉시 중단 |
| 증거 보관 | 스크린샷·로그 발췌 — **민감정보 삭제** 후 |

## 3. 방법론(요약만 — 상세 공격 기술은 기재하지 않음)

1. **자산 발견**: Lab 인벤토리와 실제 노출이 일치하는지 확인합니다.
2. **구성 검토**: 기본 비밀번호, 불필요한 포트, 과도한 권한.
3. **검증**: 발견 사항은 **재현 가능한 최소 단계**로 내부 보고서에만 기록하고, 공개 문서에는 **요약**만 둡니다.

## 5. 포트폴리오에 쓰는 문장 예시 (본인 경험으로 치환)

> “개인 샌드박스에서 ROE를 작성한 뒤, 자산 범위 내에서만 구성 취약점을 점검했습니다. 공개 저장소에는 범위·요약·재발 방지만 포함했습니다.”
