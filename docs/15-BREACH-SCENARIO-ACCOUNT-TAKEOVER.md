# 침해 시나리오 (1페이지) — 계정 탈취 가정 · 데모랩

> **가상 테이블탑**용. 실제 사건·실제 IOC 아님.

## 시나리오

파트너 관리자 계정이 **피싱**으로 유출되었다고 가정. 공격자는 MFA 없는 레거시 콘솔에 접근 시도.

## 탐지 (Detect)

| 단서 | 담당 |
|------|------|
| 동일 ASN에서 관리 콘솔 로그인 실패 급증 | 관제 (`RB-03`) |
| 비정상 국가에서의 세션 | SIEM 규칙(문서상) |

## 차단 (Contain)

| 조치 | 담당 |
|------|------|
| 의심 IP 대역 임시 차단 `WF-9921` | 보안 |
| 해당 계정 세션 무효화·비밀번호 리셋 | 플랫폼 |
| 레거시 콘솔 MFA **강제** 플래그 | IAM |

## 복구 (Recover)

- 영향 계정 목록 확정, 파트너 통지 **템플릿** 준비(법무 검토 가정).
- 포스트모템: [03번 레포 템플릿](https://github.com/yunsunghun/portfolio-03-incident-response).

## 사후 (Improve)

- MFA 적용 범위 확대, `access-review`에 레거시 콘솔 포함.
