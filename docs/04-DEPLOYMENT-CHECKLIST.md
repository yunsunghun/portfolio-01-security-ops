# 배포 · 릴리즈 체크리스트

## 배포 전 (Pre-flight)

- [ ] 변경 요약 및 롤백 계획이 티켓/PR에 링크됨.
- [ ] 스테이징 또는 카나리에서 **헬스·스모크 테스트** 통과.
- [ ] DB 마이그레이션(해당 시) 백업 및 실행 순서 문서화.
- [ ] 온콜 인수인계(배포 시간대, 연락처).

## 배포 중 (During)

- [ ] 배포 단계별 로그 보존(파이프라인 아티팩트).
- [ ] 트래픽 전환 비율/순서 준수(블루그린·롤링 등 팀 표준).

## 배포 후 (Post-deploy)

- [ ] `/health` 및 핵심 사용자 저니 1건 이상 확인.
- [ ] 에러율·지연 메트릭 기준선 대비 이상 없음(대시보드 있을 때).
- [ ] 이상 시 **롤백 실행** 여부 결정 및 기록.

## 롤백 트리거 예시 (팀 합의용)

| 조건 | 권장 조치 |
|------|-----------|
| 헬스 실패 지속 N분 | 롤백 검토 |
| 에러율이 기준의 2배 이상 | 즉시 롤백 후 RCA |
| 데이터 불일치 징후 | 트래픽 차단 + 긴급 회의 |

## 본 저장소(데모) 적용

`examples/demo-stack` 재기동 시 아래만 확인해도 됩니다.

```bash
curl -sf http://localhost:8080/health && echo OK
```
