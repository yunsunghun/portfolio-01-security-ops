# 보안 거버넌스 · 관제 프레임

## 1. 보안 목표 (CIA 관점)

| 원칙 | 운영 행위 예시 |
|------|----------------|
| 기밀성 | 최소 권한 SSH, 시크릿 미커밋, `.gitignore` 정비 |
| 무결성 | CI에서 스크립트·문서 린트, 릴리즈 태그와 아티팩트 연계 |
| 가용성 | 헬스체크, 디스크 임계치 알림, 런북 기반 복구 |

## 2. 관제 관점 체크리스트

- [ ] **자산 목록**: 서비스 이름, URL, 담당 온콜, 마지막 배포 시각.
- [ ] **로그**: 접근 로그 보존 기간, 민감 필드 마스킹 정책.
- [ ] **알림**: 심각도(Severity) 정의, 중복 알림 억제(aggregation) 원칙.
- [ ] **에스컬레이션**: 1차(온콜) → 2차(팀 리드) → 3차(임원) 연락망.
- [ ] **사후 분석**: RCA 템플릿 작성, 재발 방지 액션 아이템 추적.

## 3. 접근 통제 (서버)

1. **관리자 접근**: 공인키 인증 우선, 비밀번호 로그인 비활성화 권장.
2. **sudo**: 명령 단위 위임, 감사 로그 보존.
3. **패치**: OS·런타임 CVE 모니터링 주기 명시 (주간/월간 등 팀 합의).

## 4. 취약점 대응

- `security/templates/cve-response-template.md`를 복사해 이슈별로 기록.
- 긴급 패치 시 **변경 관리**: 긴급 CAB 예외 규칙과 사후 보고.

## 5. CI/CD 보안 (본 저장소)

- Pull Request 시 **ShellCheck**로 셸 스크립트 정적 분석.
- (선택) Secret 스캔은 조직 정책에 맞게 `gitleaks` 등 추가 권장.

## 6. 한계 명시 (포트폴리오용)

실제 SOC 연동(SIEM, SOAR), WAF 룰 세트, 침해 대응 포렌식은 **범위 밖**이며, 02·03 프로젝트에서 심화할 수 있습니다.
