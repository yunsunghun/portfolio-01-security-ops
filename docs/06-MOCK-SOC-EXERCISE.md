# 모의 관제(Tabletop / Lab) 시나리오

**가상의 알림**을 출발점으로, 관제·운영·개발이 같은 언어로 대화하는 연습용 시나리오입니다.

## 0. 사전 합의

- **심각도 정의**: P1~P4 (`docs/03-RUNBOOKS.md`와 맞출 것)
- **연락망**: 온콜 → 팀 리드 → (필요 시) 보안 담당
- **기록**: 아래 타임라인 표를 복사해 채움

## 1. 시나리오 A — “API 에러율 급증” (가상)

### 배경 (가상)

- `api-svc`가 배포된 직후, **5xx 비율**이 기준선의 3배로 상승했다는 **가상 알림**이 발생했습니다.

### 참가 역할

| 역할 | 연습할 행동 |
|------|-------------|
| 관제 | 알림 확인, 중복 억제, 티켓 번호 부여 |
| 온콜 SRE | 헬스·최근 배포·리소스 병목 확인 |
| 백엔드 | 변경 diff, 피처 플래그, DB 슬로우 쿼리 후보 |
| 보안 | 비정상 트래픽(봇넷) vs 버그 구분 가설 |

### 타임라인 기록표 (복사해서 사용)

| 시각(가상) | 사건 | 조치 | 담당 |
|------------|------|------|------|
| T+0 | 알림 수신 | 티켓 INC-2026-001 개설 | 관제 |
| T+5 | 헬스 OK, 5xx만 증가 | 최근 배포 롤백 후보 표시 | SRE |
| T+20 | 롤백 후 정상화 확인 | 포스트모템 예약 | SRE |

### 토의 질문 (면접·스터디용)

1. **롤백**을 먼저 할지, **원인 분석**을 더 할지 판단 기준은?
2. 고객 공지가 필요한 **데이터 유출 징후**는 없다고 가정했을 때, 외부 메시지는 어떻게 짧게 쓸 것인가?

## 2. 시나리오 B — “의심 로그인 시도” (가상)

### 배경 (가상)

- 관리 콘솔에서 **짧은 시간 다수 실패 로그인**이 동일 ASN에서 관측되었습니다.

### IOC 예시 표 (가상 값 — 실제와 무관)

| 유형 | 값(예시) | 신뢰도 |
|------|----------|--------|
| IP | 198.51.100.10 | 낮음~중간 |
| User-Agent | `curl/8.x` | 참고 |
| 계정 | `admin` (존재하지 않는 서비스 계정 가정) | 참고 |

### 조치 분기 (가상)

1. **WAF/방화벽** 임시 차단 후보 목록 작성(승인 하에 적용).
2. **MFA** 적용 여부, **관리자 IP 허용 목록** 검토.
3. 애플리케이션 로그와 상관해 **브루트포스 vs 오탐** 구분.

## 3. 산출물로 남기면 좋은 것

- 채워진 **타임라인 표** 1건
- **재발 방지** 액션 2개(예: 알림 임계 재조정, 배포 캔리 기본화)

실제 SOC 도구를 쓰지 않아도, **같은 순서를 문서로 재현**하는 것만으로도 학습·포트폴리오 가치가 있습니다.
