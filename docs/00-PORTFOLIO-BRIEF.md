# 포트폴리오 01 — 기획 브리프 (Portfolio Brief)

> 가상 설정 전체: [DUMMY-CANON.md](DUMMY-CANON.md)

## 1. 한 줄 정의

**데모랩(DemoLab)**의 B2B 파트너 포털·BFF 운영을 가정하고, **가용성·보안·운영 자동화**를 코드와 문서로 재현·설명하는 저장소입니다.

## 2. 대상 독자

| 독자 | 이 저장소에서 보고 싶은 것 |
|------|---------------------------|
| **인사/채용** | 역할 정의, 협업 방식, 문서화 습관, 책임 범위의 명확성 |
| **엔지니어** | 아키텍처 판단, IaC·스크립트 품질, 장애 대응·보안 거버넌스의 깊이 |

## 3. 가상 시나리오 (프로젝트 설정)

- **조직**: 데모랩 주식회사 — 플랫폼운영팀 7명.
- **서비스**: `DL-PORTAL` (정적 포털 + 헬스), `DL-BFF` (주문 API, 문서상).
- **규모**: 단일 리전 Lab, `10.50/16` 관리망 + `10.60/16` 앱 + `10.70/16` DB (문서).
- **온콜 로테이션 (더미)**: 1주 단위 — 김온콜 → 이관제 → 박SRE (표시용 이름).

## 4. 범위 (In Scope)

1. **서버 운영**: 헬스체크, 디스크·로그, 백업 스텁 + 체크리스트.
2. **서비스 운영**: 배포 전후 점검, 롤백 판단, `INC-2026-0412` 모의 사례.
3. **보안 관제**: 기준선 체크리스트, `DLAB-SEC-2026-001` 내부 추적 예시, CI 정적 점검.
4. **가상 Lab·모의 훈련**: `docs/05`~`07` — 허가된 Lab 범위 전제.

## 5. 비범위 (Out of Scope)

- 실제 프로덕션 자격 증명, 실고객·실주문 데이터.
- 특정 클라우드 벤더 **필수** 배포 (Terraform/Ansible 패턴 + 로컬 데모로 대체).

## 6. 성공 지표 (포트폴리오 관점)

- 클론 후 **15분 이내** 데모 스택 기동.
- `examples/sample-data/`의 로그로 `log_watchdog.sh` 시연 가능.
- **문서만으로** `INC-2026-0412` 역할 분담 설명 가능.

## 7. 02·03과의 역할 분리

| 번호 | 테마 |
|------|------|
| **01** | 통합 운영 + Lab·모의 관제·윤리적 모의 침해 문서 |
| **02** | 관측성·SLO (`DL-BFF` 메트릭 예시) |
| **03** | 인시던트·포스트모템 템플릿 채움본 |

## 8. 유지보수 원칙

- 시크릿은 **환경 변수 + `.env.example`** 플레이스홀더만.
- 스크립트는 POSIX + Git Bash 호환 목표.
